멀티 모델들의 아키텍쳐 분류

Motivation#

  • 저자들은 현재 연구되고있는 멀티모델들을 조사했으며 그 모델들을 분류하고 그에 따른 데이트셋과 특징들을 해당 논문에서 설명했다.

Methodology#

  • 저자들은 모델 분류를 총 A,B,C,D 4개로 분류했다. A와 B는 모델 내부에 파라미터를 추가하는 Deep fusion, C와 D는 입력층에 파라미터를 추가하는 Early fusion이다.

  • Type A : encoder를 통해 LLM에 일반적인 cross-attention Layer로 상호작용한다. 보통 self-Attention뒤 혹은 앞에 넣는지로 갈린다.

  • 장점 : 모델의 흐름을 미세하게 조절이 가능. end-to-end 학습과 출력가능

  • 단점 : 필요 학습 데이터, 모델 사이즈 증가, 사용하는 LLM의 구조에대한 이해도가 필요. cross-attention Layer추가 후 다른 Layer 추가가 힘듬. 스케일링이 어려움

  • Type B : 일반적인 cross-attention이 아닌 Custom Cross-attention Layer, Custom Learnable Layer을 쓴다. Custom Cross-attention Layer은 일반적으로 encoder을 쓰며 Custom Learnable Layer은 굳이 encoder을 필요로 하지않는다.

  • 장점 : 모델의 흐름을 미세하게 조절이 가능. end-to-end 학습과 출력가능. A type과 비교하여 다른 Layer 추가가 비교적 쉬움. 파라미터 조절이 쉬움

  • 단점 : 스케일링에 어려움이 존재. 필요 학습 데이터, 모델 사이즈 증가. text 아닌 다른 형식의 출력을 auto-regressive하게 학습하기 힘듬

  • Type C : 모델 내부에 어떤 값을 보내지 않으며 입력층을 통해서 값을 모델로 전달한다. 보통 encoder층을 이용하고 해당 출력을 특정한 길이의 값으로 바꾸기 위해 특정한 Layer을 쓴다. LLM 아키텍쳐에 변화를 주지않는다(Lora는 쓸수있음). Encoder 출력을 바꾸는 Layer에는 Reasmpler(Q-former과 비슷하지만 image와 text는 다른 self-attention층을 씀), Linear Layer, MLP, Q-former(가벼운 transformer. 텍스트, 이미지를 학습하는 하나의 query embeddings, cross-attention을 가지고있으며 이미지와 텍스트는 하나의 self-attention층을 공유), Attention Pooling등이 있다.

  • 장점 : 필요 학습 데이터, 모델 사이즈 증가가 비교적 적음. end-to-end 학습과 출력가능. LLM을 쉽게 바꿀수있음. 모델 타입의 모델 사용가능. Layer추가가 비교적 쉬움.

  • 단점 : 모델의 흐름을 미세하게 조절이 힘듬. 텍스트가 출력으로 auto-regressive하게 학습이 힘듬

  • Type D : 이미지, 비디오, 오디오를 특정한 토큰들로 변환하는 타입이다. LLM, Encder-Decder모델을 쓰는 형식으로 나뉜다.

  • 장점 : 모든 형식의 출력에 auto-regressive하게 학습이 가능. end-to-end 학습과 출력가능.

  • 단점 : 모델의 흐름을 미세하게 조절이 힘듬. 학습해야 하는 파라미터가 다른 타입들 보다 많음(다른 Token들을 학습해야 하기에)
